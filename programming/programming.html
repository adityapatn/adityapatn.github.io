My Programming Journey

I started going to coding classes in 2018, where I learned block-based programming using Scratch. I also learnt programming in Python by age 12. I have a lot of experience programming in Python, although I still have a lot to learn. I love playing the video game Minecraft, and in 2023 I started dabbling with making Minecraft mods using the programming language Java. At this point I also started learning frontend web development using HTML, CSS, and JavaScript. Programming Resources

I've provided a list below of resources I found super useful as I started programming and especially while I built this website.

  • Understanding Frontend Programming - this explanation gave me great context as to how the Internet works, and explains the basics of frontend programming.
  • What is Object-Oriented Programming? Great tutorial covering the differences between procedural programming and object-oriented programming, as well as the advantages of each one.
  • Understanding Backend Programming This gave me so much insight on how the world's biggest websites work behind the scenes, as well as how clients, servers, and databases communicate.
  • Mosh Hamedani's YouTube Channel - He covers each topic thoroughly and makes it easy to understand with great visuals.
  • All of W3Schools - it's a great resource for specific feature tutorials, as well as super helpful documentation for so many languages.
  • How to Use Git - The best tutorial for beginners I've seen so far explaining version control systems and git syntax.
  • CSS Flexbox Layout Guide - this explained CSS display properties really well for me and I refer back to it all the time.
  • How to Make your Command Line Beautiful - this changed my life. genuinely. It's not only aesthetically pleasing but also gives you useful information (and autofill!)

Programming Advice

First of all, I acknowledge that I'm not the most qualified person to give programming advice. I don't even have a StackOverflow account, so do I really have freedom of speech? That being said, I think this list will be helpful for beginning programmers (don't quit!).

  1. Search it up. Most of your questions are things other people have asked before. With the advent of the Internet, you quite literally have the world's foremost knowledge at your fingertips. Use it. Using your resources doesn't make you a bad programmer, and helps you learn so much.
  2. Don't copy-paste other people's code. I mean this loosely, but you shouldn't copy an entire program without understanding how it works. Learn how it works, then steal it (I'm joking, don't sue me).
  3. Tutorials, tutorials, tutorials! When you have a specific question, try to go broader and learn more about the subject. Even if you only need to know which command you use to make a new branch using git, covering all the git commands will make you a better programmer.
  4. Don't back away from difficult projects. If there's something you really want to make happen but you don't know how to do it, stick with it! You'll learn so much in the process and it's so satisfying when you finish a project you've been wanting to make for a while. I'm so glad I've learned enough HTML, CSS, and JS to make my own website.
  5. Write it down. When you have a difficult problem you need to solve (specific function in program, algorithm homework, interview question, etc.), write it down first. It's worth the time to think about what exactly you need the program to do and write it down before you start writing the actual program. It'll help clear up the problem and make solving it go faster.
